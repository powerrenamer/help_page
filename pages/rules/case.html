Case Rule

The case rule allows you to modify filename case in multiple ways. Here's a detailed usage guide.

Feature Overview

  • Capitalize first letter of each word in filename
  • Convert entire filename to lowercase
  • Convert entire filename to uppercase
  • Invert case of each character
  • Capitalize only first letter
  • Apply sentence case (capitalize first letter of each sentence)
  • Apply case changes to specific text segments
  • Control file extension handling separately

Usage Steps

1. Choose Case Type

Title Case

Convert first letter of each word to uppercase

my_file_name.txt → My_File_Name.txt

Lowercase

Convert all characters to lowercase

MyFileName.TXT → myfilename.txt

Uppercase

Convert all characters to uppercase

my_file.txt → MY_FILE.txt

Invert Case

Convert uppercase to lowercase and lowercase to uppercase

MyFile.txt → mYfILE.txt

First Letter Only

Capitalize only the first letter of the filename

my_file.txt → My_file.txt

Sentence Case

Capitalize first letter of each sentence

my file name.txt → My file name.txt

2. Configure Target Segments

Apply to Specific Segments

When enabled, specified text segments maintain their case regardless of other options

Use commas to separate multiple text segments

Example (using segments "MY,Is,jack"):

my_name_is_Jack.mp3 → MY_name_Is_jack.MP3

3. Set Extension Options

Ignore Extension

When enabled, case changes won't affect file extension

myfile.txt → MYFILE.txt

Lowercase Extension

Force file extension to lowercase

myFile.TXT → myFile.txt

Uppercase Extension

Force file extension to uppercase

myFile.txt → myFile.TXT

Important Notes

Extension Handling

  • Extension options take precedence over main case conversion
  • Only one extension case option (lowercase or uppercase) can be active at a time

Target Segment Processing

  • Multiple segments should be comma-separated
  • Segments are processed in the order listed

Special Characters

  • Case conversion only affects letters
