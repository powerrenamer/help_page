Wildcards

Wildcards allow you to use simple pattern matching symbols for file renaming operations. Here's a detailed explanation and usage guide for wildcard rules.

Wildcard Patterns

Asterisk (*)

Matches any number of characters (including zero characters)

  • *.txt - Matches all text files: file.txt, document.txt, note.txt
  • doc* - Matches all files starting with "doc": doc.txt, document.pdf
  • *2025* - Matches files containing "2025": report2025.doc, 2025tax.pdf

Question Mark (?)

Matches any single character

  • file?.txt - Matches: file1.txt, fileA.txt, but not file10.txt
  • ????.txt - Matches any txt file with exactly 4 characters in the name
  • doc-?.pdf - Matches: doc-1.pdf, doc-A.pdf

Square Brackets ([ ])

Matches any single character within the brackets

  • file[123].txt - Matches: file1.txt, file2.txt, file3.txt
  • doc[AB].pdf - Matches: docA.pdf, docB.pdf
  • image[0-9].jpg - Matches: image0.jpg through image9.jpg

Usage Tips

  • * is best for matching text of variable length
  • ? is best used when you know the exact position and length
  • [ ] is best for limited choices at a specific position

Important Notes

  • Start with simple patterns and gradually add complex rules
  • Use the preview feature to verify matching results
  • Test with a small set of files before processing large batches
  • Be aware of unexpected matches that may occur with wildcard combinations
