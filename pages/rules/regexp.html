Regular Expression Rule

The regular expression rule allows you to use powerful pattern matching and replacement features for renaming files. Here's a detailed usage guide.

Feature Overview

  • Use regular expressions for pattern matching
  • Support advanced text replacement and backreferences
  • Support case sensitivity options
  • Support extension protection
  • Insert metadata tags

Usage Steps

1. Configure Search and Replace

In the "Search and Replace" section:

  • Regular Expression: Enter the search pattern
    • \\d+ - Match any sequence of numbers
    • (old)-(new) - Capture two groups separated by a hyphen
  • Replacement Text: Specify the replacement pattern
    • Use $1 , $2 etc. to reference captured groups
    • Example: Replacing (old)-(new) with $2_$1 swaps positions and changes the separator

2. Configure Options

  • Case Sensitive: When enabled, pattern matching considers letter case
    • Enabled: [A-Z]+ matches only uppercase letters
    • Disabled: [A-Z]+ matches both upper and lowercase letters
  • Ignore Extension: When enabled, pattern matching excludes file extension
    • For file.name.txt , pattern applies only to file.name

Regular Expression Syntax

  • \\d - Match digits
  • \\w - Match letters, digits, underscores
  • \\s - Match whitespace characters
  • [...] - Match character set
  • (...) - Capture group
  • ? , * , + - Quantifiers

Important Notes

  • Test patterns on a small set of files first
  • Use the preview feature to verify changes
  • Be careful not to create invalid filenames
  • Avoid patterns that might create duplicate filenames
  • Consider breaking complex patterns into steps

Backreference Guide

  • Use $1 through $9 to reference captured groups
  • Capture groups are numbered from left to right by opening parentheses
