Replace Rule

The replace rule allows you to search for specific text patterns in filenames and replace them with new text. Here's a detailed usage guide.

Feature Overview

  • Replace all occurrences of specified text
  • Replace only the first occurrence of specified text
  • Replace only the last occurrence of specified text
  • Support case-sensitive matching
  • Support whole word matching
  • Support wildcard patterns with backreferences
  • Support preserving file extensions during replacement
  • Insert metadata tags

Usage Steps

1. Configure Search and Replace

Search Text

Enter the text pattern to find in filenames

Replace Text

Enter the text to replace matching patterns

When wildcards are enabled, you can use special patterns and backreferences:

Search: file(*) Replace: document$1

Will convert file123.txt to document123.txt

2. Choose Operation Type

All

Replace every matching text

old_file_old.txt → new_file_new.txt

First

Replace only the first occurrence of matching text

old_file_old.txt → new_file_old.txt

Last

Replace only the last occurrence of matching text

old_file_old.txt → old_file_new.txt

3. Configure Options

Case Sensitive

When enabled, text matching will be case sensitive

"File" will not match "file" in filenames when enabled

Whole Word Only

When enabled, only matches complete word segments (disabled when using wildcards)

my_file.txt → my_file.txt (won't match "file") my file name.txt (will match "file")

Ignore Extension

When enabled, replacement only affects the filename, keeping the extension unchanged

Use Wildcards

When enabled, special characters are treated as wildcards, and $n as backreferences:

  • ? matches any single character
  • * matches zero or more characters
  • [] matches any character within brackets
  • $1 , $2 etc. reference captured groups in order

Important Notes

Wildcards and Backreferences

  • When using wildcards, you can use $1, $2 etc. in replacement text to reference captured groups
  • Ensure wildcard patterns correctly capture the required text portions

Extension Handling

  • When "Ignore Extension" is enabled, replacement won't affect file extensions
  • Recommended to enable this option when extension modification is not needed

Case Sensitivity

  • By default, text matching is case insensitive
  • Enable "Case Sensitive" option when exact case matching is required

Whole Word Matching

  • Whole word option prevents partial matches
  • This option is not available when using wildcards

Pattern Safety

  • When using wildcards and backreferences, test patterns on a small set of files first
  • Ensure replacement results meet expectations before applying to many files
