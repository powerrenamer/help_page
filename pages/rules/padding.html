Fill Rule

The fill rule allows you to pad filenames to a specific length using specified characters. This rule is particularly useful when you need to standardize filename lengths or add leading characters.

Feature Overview

  • Fill filenames with specified characters
  • Support left and right padding
  • Set target length
  • Support ignoring extensions
  • Support filling only numeric parts

Usage Steps

1. Set Fill Parameters

Fill Character

Choose the character for padding, common options:

  • Number 0: for numeric sequences
  • Custom characters

Target Length

Set the total length after filling

1.jpg → 001.jpg (Set length to 3, using 0 as fill character)

2. Choose Fill Mode

Left Fill

Add fill characters "a" to the left side of filename, length 6

99.jpg → aaaa99.jpg

Right Fill

Add fill characters "_" to the right side of filename, length 6

file.jpg → file__.jpg

Zero Fill

Only fill numeric parts in filename, length 2

photo5.jpg → photo05.jpg

3. Configure Fill Options

Ignore Extension

When enabled, fill length calculation excludes extension

1.jpg → 001.jpg (Length 3 excludes .jpg)

Keep Original Length

If filename already meets or exceeds target length, keep it unchanged

1000.jpg → 1000.jpg (Zero fill, length 4, already meets length 4, remains unchanged)

Important Notes

Length Calculation

  • Target length includes fill characters and original filename
  • Extension length is not counted when "Ignore Extension" is enabled

Fill Characters

  • Avoid characters that might make filenames invalid
  • Consider file system restrictions on special characters
