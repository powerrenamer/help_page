Remove Rule

The remove rule allows you to remove specific text patterns from filenames. Here's a detailed usage guide.

Feature Overview

  • Remove all occurrences of specified text
  • Remove only the first occurrence of specified text
  • Remove only the last occurrence of specified text
  • Support case-sensitive matching
  • Support whole word matching
  • Support wildcard patterns using ? , * , [ and ]
  • Support ignoring extensions

Usage Steps

1. Set Text to Remove

In the "Remove Text" section, enter the text you want to remove from filenames.

  • Example: To remove the text "copy" from document_copy.txt , enter "copy"
  • When wildcards are enabled, you can use patterns like "copy*" to match "copy1", "copy2", etc.

2. Choose Operation Type

Remove All

Remove every occurrence of the specified text

copy_document_copy.txt → _document_.txt

Remove First

Remove only the first occurrence of the specified text

copy_document_copy.txt → _document_copy.txt

Remove Last

Remove only the last occurrence of the specified text

copy_document_copy.txt → copy_document_.txt

3. Configure Additional Options

Case Sensitive

When enabled, text matching will be case sensitive

Removing "Copy" will not match "copy" in filenames

Whole Word Only

When enabled, only matches complete word segments (disabled when using wildcards)

my_copy my copy.txt → my_copy my .txt (matches "copy", but not "my_copy")

Ignore Extension

When enabled, removal only affects the filename, keeping the extension unchanged

Use Wildcards

When enabled, special characters are treated as wildcards:

  • ? matches any single character
  • * matches multiple characters
  • [] matches any character within brackets, e.g., [0-9], [a-z], [A-Z], [abc123] match any digit, lowercase letter, uppercase letter, or any character from "abc123"

Important Notes

Wildcard Usage

  • When wildcards are enabled, the whole word matching option is automatically disabled
  • Be careful with wildcard patterns as they may match more text than expected

Extension Handling

  • When "Ignore Extension" is enabled, removal won't affect file extensions
  • Recommended to enable this option when extension modification is not needed

Case Sensitivity

  • By default, text matching is case insensitive
  • Enable "Case Sensitive" option when exact case matching is required

Whole Word Matching

  • Whole word option prevents partial matches
  • This option is not available when using wildcards
